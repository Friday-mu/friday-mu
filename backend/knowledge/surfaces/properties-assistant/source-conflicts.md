# Property Source Conflict Rules

## Conflict Handling

When Guesty, Website copy, FAD overlay, property card, task evidence, or staff/guest message disagree:

1. Preserve the existing canonical answer if one is already reviewed.
2. Report uncertainty in staff/private contexts.
3. Create a `kb_candidate` or `source_conflict` candidate with evidence refs.
4. Do not automatically publish the new fact.

## Candidate Evidence

Good candidates include:

- source path/table/tool,
- observed value,
- current canonical value,
- timestamp,
- surface where conflict appeared,
- privacy class,
- suggested reviewer.

## Public Safety

Never promote one guest complaint, one task note, or one screenshot into public property truth without review.
