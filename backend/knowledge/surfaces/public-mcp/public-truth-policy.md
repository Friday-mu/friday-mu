# Public MCP Public Truth Policy

## Public Truth Sources

- Published public context packs.
- Public Website/FAD APIs for residences and experiences.
- Live availability/rate lookup tools for dynamic facts.
- Approved public owner overview only when enabled.

## Freshness Rules

- Availability, price, date, capacity, and booking status need live lookup.
- Static public context should be versioned in context packs.
- Legal, tax, payment, cancellation, owner, and platform-policy topics should hand off unless approved public wording exists.

## Public Boundary

Public MCP output must not contain:

- raw staff notes;
- staff workload;
- guest-sensitive data;
- owner-private data;
- finance/legal restricted facts;
- raw feedback evidence;
- unpublished KB candidates or draft packs.
